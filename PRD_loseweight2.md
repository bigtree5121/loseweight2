# 产品需求文档（PRD）：轻记 · 极简体重记录工具

| 版本 | 日期 | 作者 | 变更内容 |
|------|------|------|----------|
| v1.0 | 2026-04-03 | Product Owner | 初始版本 |
| v1.1 | 2026-04-03 | Product Owner | 新增曲线对比功能（目标体重 vs 实际体重） |

## 1. 产品概述

### 1.1 背景
用户需要一个极简风格的减肥记录网页，核心是**按日期设定目标**，并在目标日期当天或之后手动录入**实际体重、运动情况、饮食情况、护肤情况**。产品不追求社交、复杂图表或智能分析，仅聚焦于“计划 vs 实际”的纯粹记录体验。

### 1.2 定位
- **极简工具**：无广告、无社区、无推送。
- **数据私有**：所有数据保存在用户浏览器本地（LocalStorage），不上传任何服务器。
- **离线可用**：一次加载后无需网络即可使用。

### 1.3 目标用户
- 希望用最简单的方式追踪每日减重计划与实际执行情况的用户。
- 对数据隐私敏感，不想注册账号的用户。
- 喜欢手账式记录、自己掌控节奏的用户。

## 2. 用户核心故事

- 作为用户，我可以在日历上选择任意一个**未来的或过去的日期**，为该日期设定一个**目标体重**（例如：希望在4月10日体重达到60kg）。
- 作为用户，我可以在已经设定目标的日期里，录入那一天的**实际体重**、**运动情况**（文字）、**饮食情况**（文字）、**护肤情况**（文字）。
- 作为用户，我希望一眼看出**哪些日期已经设定了目标**，**哪些日期已经完成了实际记录**。
- 作为用户，我希望随时**修改或删除**某个日期的目标或实际记录。
- **【新增】作为用户，我希望通过曲线图直观对比“预设目标体重”与“实际体重”的变化趋势，看清自己与目标的差距。**

## 3. 功能需求

### 3.1 目标管理（按日期）

| 功能点 | 描述 | 交互方式 |
|--------|------|----------|
| 添加目标 | 选择一个日期，输入该日期的目标体重（单位：kg，支持小数点后1位）。 | 点击日历上的空白日期 → 弹出浮层 → 输入体重 → 保存。 |
| 查看目标 | 日历上，已设定目标的日期显示**小圆点**或**目标体重数字**（可配置显示方式）。 | 鼠标悬浮或点击日期显示详细目标卡片。 |
| 修改目标 | 重新点击已设定目标的日期 → 编辑体重数值 → 保存。 | 同添加交互。 |
| 删除目标 | 在目标卡片或编辑界面提供“删除目标”按钮。 | 删除后该日期恢复空白状态。 |

> **说明**：一个日期只能有一个目标体重。如果用户想设定多维度目标（如运动目标、饮食目标），本版本暂不做，保持极简。

### 3.2 实际记录（手动录入）

在**已经设定目标**的日期上，用户可以录入以下四项实际数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| 实际体重 | 数字（kg） | 支持小数点后1位，必填（但可留空表示未称重）。 |
| 运动情况 | 短文本 | 例如：“跑步30分钟+拉伸”。上限100字，手动输入。 |
| 饮食情况 | 短文本 | 例如：“早餐全麦面包，午餐鸡胸沙拉”。上限100字。 |
| 护肤情况 | 短文本 | 例如：“早C晚A，敷了面膜”。上限100字。 |

**交互流程**：
- 点击日历上**已有目标**的日期 → 弹出记录面板。
- 面板显示四个输入框（或文本域），以及“保存记录”按钮。
- 用户填写任意一个或多个字段后保存。
- 已保存实际记录的日期，在日历上用**不同颜色或勾选图标**标记（例如：绿色勾）。

**编辑与删除**：
- 再次点击同一日期，可修改或清空任意字段。
- 提供“清空当日所有记录”按钮（谨慎操作，可二次确认）。

### 3.3 日历视图（核心界面）

- **展示形式**：标准月日历（周一或周日起始可选，默认周一）。
- **导航**：支持按年/月切换（上一月/下一月/回到今日）。
- **标记规则**：
  - 仅设定目标、未记录实际 → 浅灰色圆点。
  - 已记录实际（至少填了一项） → 绿色对勾或实心圆点。
- **点击日期行为**：
  - 无目标 → 弹出“添加目标”面板。
  - 有目标 → 弹出“实际记录”面板（同时显示目标体重供参考）。

### 3.4 列表视图（辅助）

为方便快速浏览和编辑，提供**列表模式**（开关切换）：
- 按日期倒序排列所有**已设定目标**的记录。
- 每一行显示：日期、目标体重、实际体重、运动/饮食/护肤摘要（可缩略）。
- 点击某一行进入编辑（同日历弹窗）。

### 3.5 【新增】曲线对比功能（目标 vs 实际）

#### 3.5.1 概述
在日历视图下方或独立区域，提供一张**折线图**，同时绘制两条曲线：
- **预设目标曲线**：根据用户设定的所有目标体重（按日期升序连接）。
- **历史实际曲线**：根据用户录入的实际体重（按日期升序连接）。

用户可直观看到每个日期上“实际体重”偏离“目标体重”的程度，以及整体趋势是否达标。

#### 3.5.2 数据来源与规则
- **横轴**：日期。只显示**同时存在目标体重和实际体重**的日期（确保可比性）。若某日期有目标但无实际体重，则该日期不参与曲线绘制（避免断点）。
- **纵轴**：体重（kg），刻度范围根据所有数据的 min/max 自动扩展，留出 10% 边距。
- **曲线样式**：
  - 预设目标曲线：**虚线**，颜色为灰色（#9CA3AF）或淡蓝色（#3B82F6），线宽 2px。
  - 历史实际曲线：**实线**，颜色为绿色（#2C7A4D），线宽 2px。
- **数据点标记**：
  - 每个日期位置显示一个小圆点（目标为灰色空心圆，实际为绿色实心圆）。
  - 鼠标悬浮（或移动端长按）时显示 tooltip，展示“日期，目标体重，实际体重”。

#### 3.5.3 交互与布局
- **位置**：放在日历视图下方，与日历之间留出 24px 间距。
- **尺寸**：宽度 100%，高度 200px ~ 250px（移动端可适当压缩）。
- **缩放与滚动**：图表支持水平滑动查看较长时间范围（如果日期跨度超过 30 天）。默认显示**最近 30 天**的数据（从今天往前推），并提供“显示全部”按钮或日期范围选择器（极简原则下，v1.1 仅提供“最近30天/全部”切换）。
- **无数据状态**：如果用户未录入任何实际体重，图表区域显示灰色提示文字：“暂无实际体重数据，请在日历中记录体重后查看曲线”。
- **与日历联动**：当用户在日历中修改/删除目标或实际体重后，图表自动刷新。

#### 3.5.4 技术实现要求
- 使用轻量级图表库（如 `Chart.js`、`ECharts` 或纯 SVG 绘制），但必须保证首次加载体积小巧。
- 若无外部依赖，可基于 Canvas 自行绘制折线图（推荐做法，完全自主控制）。
- 图表必须响应窗口 resize 事件，重新绘制。

#### 3.5.5 边界情况
| 场景 | 处理方式 |
|------|----------|
| 仅有目标体重，无任何实际体重 | 图表显示提示，不画任何曲线。 |
| 实际体重记录日期不连续 | 按实际存在的日期顺序连线，跳过无数据的日期（不强行连接）。 |
| 目标体重在某些日期缺失（用户未设目标） | 曲线只绘制有目标且同时有实际的日期，缺失日期不插入估算值。 |
| 所有数据点体重相同 | 纵轴仍显示合理范围（如 60±2kg）。 |

## 4. 数据模型（前端存储）

使用 `localStorage`，数据结构如下：

```json
{
  "targets": {
    "2026-04-10": { "targetWeight": 60.0 },
    "2026-04-15": { "targetWeight": 59.5 }
  },
  "records": {
    "2026-04-10": {
      "actualWeight": 60.2,
      "exercise": "跑步5km",
      "diet": "低碳日",
      "skinCare": "保湿面膜"
    },
    "2026-04-12": {
      "actualWeight": 59.8,
      "exercise": "休息",
      "diet": "正常",
      "skinCare": null
    }
  }
}
