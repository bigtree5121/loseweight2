# 待办事项应用部署指南

## 部署到GitHub Pages的步骤

### 1. 安装Git

如果您的电脑还没有安装Git，请先下载并安装：
- 访问 [https://git-scm.com/downloads](https://git-scm.com/downloads)
- 下载适合您操作系统的版本并安装
- 安装完成后，打开命令提示符或终端，运行 `git --version` 确认安装成功

### 2. 创建GitHub账号

如果您还没有GitHub账号：
- 访问 [https://github.com/](https://github.com/)
- 点击"Sign up"创建账号
- 完成邮箱验证和设置

### 3. 创建新的GitHub仓库

- 登录GitHub
- 点击右上角的"+"按钮，选择"New repository"
- 输入仓库名称（例如：todo-app）
- 选择"Public"（公开）
- 勾选"Add a README file"
- 点击"Create repository"

### 4. 初始化本地Git仓库并推送代码

打开命令提示符或终端，进入 `d:\vibecoding\test3` 目录：

```bash
# 进入目录
cd d:\vibecoding\test3

# 初始化Git仓库
git init

# 配置用户名和邮箱
git config user.name "您的GitHub用户名"
git config user.email "您的GitHub邮箱"

# 添加文件到暂存区
git add .

# 提交更改
git commit -m "Initial commit"

# 添加远程仓库（将YOUR_USERNAME替换为您的GitHub用户名，YOUR_REPO替换为仓库名称）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# 推送到GitHub
git push -u origin main
```

### 5. 启用GitHub Pages

- 进入您的GitHub仓库页面
- 点击"Settings"选项卡
- 在左侧菜单中点击"Pages"
- 在"Source"部分，选择"main"分支，然后选择"/ (root)"
- 点击"Save"
- 等待几分钟，GitHub Pages会生成部署链接

### 6. 访问您的应用

部署完成后，您可以通过 `https://YOUR_USERNAME.github.io/YOUR_REPO` 访问您的待办事项应用。

## 其他部署选项

### Vercel
1. 访问 [https://vercel.com/](https://vercel.com/)
2. 登录或注册账号
3. 点击"Add New Project"
4. 选择"Import Project"
5. 选择"Import from Git Repository"并连接您的GitHub仓库
6. 点击"Deploy"按钮
7. 部署完成后，Vercel会提供一个访问链接

### Netlify
1. 访问 [https://www.netlify.com/](https://www.netlify.com/)
2. 登录或注册账号
3. 点击"Add new site" -> "Deploy manually"
4. 拖拽 `d:\vibecoding\test3` 目录到上传区域
5. 等待部署完成，Netlify会提供一个访问链接

## 部署后测试

部署完成后，请测试以下功能：
- 添加新任务
- 标记任务为完成/未完成
- 删除任务
- 刷新页面后数据是否保留
- 在不同设备上的响应式显示

如果遇到任何问题，请检查文件路径和部署设置是否正确。
